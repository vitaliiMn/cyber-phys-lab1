# Лабораторная работа №1 — Обнаружение объектов (Fire Detection)

## Описание задачи
Автоматическое обнаружение очагов возгорания на видеопотоке с камер наблюдения для систем раннего оповещения.

## Датасет
**Набор данных:** [Fire-detection (Kaggle)](https://www.kaggle.com/datasets/metinmekiabullrahman/fire-detection)

**Характеристики:**
- ~754 изображения с разметкой bounding boxes
- 1 класс: `fire`
- Формат: YOLO (отдельные папки images/labels для train/valid/test)
- Уникальный специализированный датасет (не COCO/Pascal VOC)

**Обоснование выбора:**
- Прямая привязка к практической задаче мониторинга пожарной безопасности
- Высокий визуальный контраст объектов (огонь хорошо различим)
- Разнообразие сценариев (домашние пожары, промышленные, открытое пламя)

## Метрики качества
| Метрика | Обоснование |
|---------|-------------|
| **mAP@0.5** | Mean Average Precision при IoU=0.5. Основной порог срабатывания тревоги — достаточно для фиксации факта наличия огня |
| **mAP@0.5:0.95** | Средний mAP при IoU от 0.5 до 0.95. Оценивает точность локализации bounding box для интеграции с системами пожаротушения |
| **Precision** | Доля правильных срабатываний. Минимизирует ложные тревоги (тени, блики, оранжевые объекты) |
| **Recall** | Доля найденных объектов. Минимизирует пропуски реальных очагов — критично для безопасности |

## Сравнение результатов: Baseline vs Improved

### Таблица метрик

Обучение проводилось на 15 эпохах, размер изображения 320×320, валидация на тестовой выборке (754 изображения).

| Метрика | Baseline (YOLOv11n, SGD) | Improved (AdamW, Augs, HSV) | Δ (Improved − Baseline) |
|---------|-------------------------|----------------------------|------------------------|
| **mAP@0.5** | **0.8755** | 0.8568 | **−0.0187** ↓ |
| **mAP@0.5:0.95** | **0.6103** | 0.5892 | **−0.0211** ↓ |
| **Precision** | **0.8231** | 0.8220 | −0.0011 ↓ |
| **Recall** | **0.7790** | 0.7747 | −0.0043 ↓ |

### Анализ различий

| Гипотеза улучшения | Ожидаемый эффект | Фактический результат | Объяснение |
|-------------------|-----------------|----------------------|------------|
| **AdamW вместо SGD** | Более стабильная сходимость, меньшие колебания loss | Небольшое снижение всех метрик | AdamW с `lr0=0.001` требует больше эпох для сходимости; при 15 эпохах SGD показал себя эффективнее на данном датасете |
| **Mosaic (0.5) + MixUp (0.1)** | Повышение робастности к масштабу и наложениям объектов | Снижение mAP@0.5 на 1.9% | Для чётких, высококонтрастных объектов (огонь) агрессивные аугментации вносят избыточный шум и искажают естественные границы пламени |
| **HSV-сдвиги (h=0.015, s=0.7, v=0.4)** | Устойчивость к изменению освещения и цветового баланса | Небольшое снижение Recall | Цвет огня (оранжевый/жёлтый/красный) — ключевой признак; искусственные сдвиги тона/насыщенности затрудняют обучение модели |
| **Weight decay + lower LR** | Снижение переобучения, лучшая генерализация | Метрики стабильны, но не улучшились | Датасет небольшой (~754 изображения), регуляризация полезна, но не компенсирует негативный эффект от аугментаций |



### Ключевые выводы

1. **Базовая конфигурация оказалась оптимальной** для данной задачи:
   - mAP@0.5 = 0.875 — высокий уровень обнаружения огня
   - Precision = 0.82 — приемлемый уровень ложных срабатываний
   - Recall = 0.78 — модель находит ~78% реальных очагов

2. **Улучшения не дали прироста** по следующим причинам:
   - Датасет содержит чёткие, контрастные объекты, для которых агрессивные аугментации избыточны
   - Малый размер выборки (~754 изображения) не позволяет полноценно использовать сложные техники регуляризации
   - Оптимизатор AdamW с низким learning rate требует больше итераций для сходимости
